Deployment Guide — Non-Technical Step-by-Step Instructions

⏱
Total Time
~30 minutes
💰
Cost
Free (both tools)
🔧
Technical Skill
Beginner friendly
Before you begin: You'll need two free accounts — one on GitHub (to host the code) and one on Supabase (the database). Both are free. No credit card needed.

1

Set Up Your Database (Supabase) Free

Estimated time: 10 minutes
a
Create a Supabase account
Go to supabase.com and click Start your project. Sign up with your GitHub or Google account — it's free.
b
Create a new project
Click New Project. Fill in:
  • Name: pricemaster-hub (or any name you like)
  • Database Password: choose a strong password and save it somewhere safe
  • Region: choose the one closest to your team
Click Create new project and wait ~2 minutes for it to be ready.
c
Run the database setup script
In the left sidebar, click SQL Editor, then click + New query.

Open the file supabase_schema.sql from the project folder, copy all of its contents, paste it into the SQL Editor, and click Run (or press Ctrl+Enter).

You should see a green "Success" message. This creates all the tables the app needs.
Tip: If you see an error saying a table already exists, that's fine — just proceed.
d
Copy your API credentials
In the left sidebar, go to Settings → API.

You'll see two values — copy both and keep them handy:
Project URL: https://xxxxxxxxxxxxxx.supabase.co Anon/Public Key: eyJhbGciOi... (a long string)
Important: The "anon key" is safe to use in a web app. Never share your "service_role" key.
2

Put the Code on GitHub Free

Estimated time: 5 minutes
a
Create a GitHub account (if you don't have one)
Go to github.com and sign up. It's free.
b
Create a new repository
Click the + icon (top-right) → New repository.
Name it pricemaster-hub, set it to Private, and click Create repository.
c
Upload the project files
On the new repository page, click uploading an existing file.
Drag the entire pricemaster-hub project folder contents (all files and folders) into the upload area.
Add a commit message like "Initial upload" and click Commit changes.
Make sure .env.local is NOT uploaded (it contains your secrets). The .gitignore file already prevents this if you use Git from the command line, but if uploading manually, just don't include it.
3

Deploy the App (Netlify — Recommended) Free

Estimated time: 10 minutes
Why Netlify? It connects directly to GitHub, builds your app automatically, and lets you set secret environment variables safely — all for free.

a
Create a Netlify account
Go to netlify.com → click Sign up → choose Sign up with GitHub.
b
Connect your GitHub repository
Click Add new site → Import an existing project → GitHub.
Authorize Netlify to access your GitHub, then select the pricemaster-hub repository.
c
Configure build settings
Netlify may auto-detect these. Confirm they are set as follows:
Build command: npm run build Publish directory: build
d
Add your Supabase credentials as environment variables
Before clicking Deploy, click Add environment variables (or go to Site settings → Environment variables after deployment).

Add these two variables using the values you copied in Step 1d:
Key: REACT_APP_SUPABASE_URL Value: https://xxxxxxxxxxxxxx.supabase.co Key: REACT_APP_SUPABASE_ANON_KEY Value: eyJhbGciOi... (your anon key)
Important: The variable names must be typed exactly as shown — including REACT_APP_ at the start.
e
Deploy!
Click Deploy site. Netlify will build the app (takes about 2–3 minutes).

When it's done, you'll see a green "Published" status and a URL like https://random-name.netlify.app.

Open that URL — your app is live! 🎉
f
Optional: Set a custom domain name
In Netlify, go to Site settings → Domain management → Add custom domain.
You can use a domain you own, or keep the free .netlify.app URL.
4

Share with Your Team

Estimated time: 2 minutes
a
Send the URL to your team
Everyone on your team just needs the Netlify URL (e.g. https://pricemaster-hub.netlify.app). No installation needed — they open it in any browser.
b
All team members share the same data
The app is set up so all team members see and edit the same projects in real time. There's no login required — the shared database does all the work.
c
Create your first real project
On first load, a demo project "Acme Capital" is created automatically. Click + NEW in the sidebar to create your first real client project. The demo project can be ignored or used as a reference.
5

Updating the App in Future

Whenever code changes are needed
1
Upload the new files to GitHub
Go to your GitHub repository, upload the updated files (you can replace existing ones).
2
Netlify redeploys automatically
Netlify watches your GitHub repository. Every time you push new code, it automatically rebuilds and redeploys within 2–3 minutes. No manual action needed.
Your data is safe: Code updates never affect your Supabase database. All projects, tasks, and RAID items remain intact.

Troubleshooting

Common issues and fixes
!
App loads but shows "Failed to load data"
Your Supabase credentials are missing or incorrect in Netlify. Go to Netlify → Site settings → Environment variables. Check that REACT_APP_SUPABASE_URL and REACT_APP_SUPABASE_ANON_KEY are spelled exactly right and have the correct values. Then trigger a redeploy (Deploys → Trigger deploy).
!
Netlify build fails with an error
Check that all files from the project folder were uploaded to GitHub — especially package.json and the src/ folder. Also make sure the build command is set to npm run build.
!
Data not saving between sessions
The app saves everything to Supabase. If data disappears, confirm the SQL schema was run successfully. In Supabase, go to Table Editor and check that tables like projects and tasks exist.
